# Testing guide

How DeckCheck is tested, where each kind of test lives, and the gates a
change has to pass.

## Layout

Two kinds of tests, split by build tag:

- **`make unit-test`** runs the untagged tests under `internal/`:
  pure domain suites (`project`, `projectfile`, `dataset`, `db`,
  `export`, `picker`, `usererror`, `translations`) that never start
  the Fyne driver.
- **`make integration-test`** runs everything tagged
  `//go:build integration`: the view and widget suites under
  `internal/ui/` plus the `fyneui` widget packages. These exercise
  real widgets through Fyne's headless test driver: `test.NewApp()`,
  then `test.Tap`, typed keys, and menu actions against the actual
  view tree. No display server is needed.

Run them with:

```bash
make unit-test         # untagged domain tests, race detector on
make integration-test  # tagged Fyne-driver tests
make test              # both
```

## Conventions

- Tests are behaviour-driven: they drive the UI the way a user would
  (tap a button, press `1`, walk the wizard) and assert on what is
  visible, not on internals. Given/When/Then comments mark the phases.
- Test names are sentence-camel: `TestWizardNavigatesBackAndRetainsInput`.
- `testify/require` only; a failed step aborts the test rather than
  cascading.
- Widget-tree helpers (find a button by label, type into an entry by
  placeholder, assert visible text) live in `internal/fynetest` and are
  shared by every UI suite. Don't re-implement tree walking locally.
- Mocks are generated by mockery from `.mockery.yml` into the
  `mocks` packages; `NewX(t)` mocks auto-assert their expectations on
  cleanup.
- Background work is synchronised, never slept on. Views own a
  `lifecycle.Lifecycle`; tests wait for quiescence through the
  `WaitForPendingOperations` shims exported from `export_test.go`
  files. There is no `time.Sleep` in the suite.
- External test packages (`foo_test`) are the rule; the only
  white-box tests are the three seams unreachable from the public
  API (`projectfile`'s atomic file swap, `db`'s migration rollback,
  `translations`' corrupt-catalog fixture).
- Two source-scanning gates keep conventions honest: the `usererror`
  codes test fails on duplicate or resurrected support codes, and the
  `translations` catalog test fails when a `lang.L` key or error-copy
  string is missing from `en.json`.

## Coverage gate

```bash
make coverage
```

runs everything with `-race -tags=integration` and fails if any
package, or the total, drops below `COVER_MIN` (80%). Generated
mocks, the `fynetest` helpers, and pure composition glue (`main`,
`dependencies`) are excluded; see `COVER_PACKAGES` in the Makefile.

## CI

`.github/workflows/ci.yml` runs the suite on Linux, macOS, and
Windows for every push and pull request, plus format check, lint
(`golangci-lint`), and the coverage gate on Linux. The release
workflow packages from the same commit only after CI is green, unless
it is dispatched with the `skip_tests` hotfix input.

## Screenshots

`make screenshots` software-renders the three screens into
`docs/screenshots/` using the same Fyne test canvas the UI tests use;
regenerate them whenever a visible screen changes.
